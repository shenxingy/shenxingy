# Design System — `shenxingy/shenxingy`

> Scope: the GitHub profile README. This file is the single source of truth for all visual decisions on the profile. Any change to the README's look must be reflected here first.

**Aesthetic**: Editorial Minimal — newspaper masthead meets type specimen.
**Differentiation** (the one thing to remember): a hairline-ruled **masthead wordmark + volume stamp**, treating the profile as if it were Issue N of a personal publication.

---

## 1. Rendering Constraints (read first)

GitHub renders profile READMEs through a sanitized markdown pipeline. The following are **the binding rules** for this system — every token below is chosen to survive them.

| Allowed                                           | Stripped / unusable          |
|---------------------------------------------------|------------------------------|
| `<a>` `<img>` `<picture>` `<div align=>` `<br>` `<hr>` | `<svg>` (inline — must be external file) |
| `<samp>` `<sub>` `<sup>` `<kbd>` `<code>`          | `<style>` `<script>`         |
| `<table>` `<details>` `<summary>`                 | Custom CSS, class attrs      |
| External SVGs via `<img src="assets/*.svg">`      | Web fonts on the body        |
| `<picture>` with `prefers-color-scheme` media     | JavaScript of any kind       |

**Implication**: all custom typography, color, and ornament must be delivered as `.svg` files committed to `assets/`, referenced by `<img>`. Body text falls back to GitHub's default system font stack — we do not fight it.

---

## 2. Typography

Two surfaces: **SVG-rendered display type** (masthead, section numerals, wordmarks) and **markdown body type** (inherits GitHub's default).

### Display (inside SVGs)

- **Primary serif** — `font-family="ui-serif, 'Iowan Old Style', 'Apple Garamond', Baskerville, 'Times New Roman', Georgia, serif"`
  - High-contrast serif fallback chain. Lands on Iowan on macOS, Cambria/Georgia on Windows, DejaVu Serif on Linux — all characterful, none generic.
- **Secondary small-caps label** — same stack, `font-variant="small-caps"` or hand-set uppercase at tracked letter-spacing `0.22em`.
- **Numeric / issue stamp** — `font-family="ui-monospace, 'SF Mono', 'JetBrains Mono', Menlo, Consolas, monospace"`, all-caps, tracked `0.16em`. The monospace against the serif creates the editorial rhythm.

### Body (markdown default)

- **Paragraph / links** — GitHub's sanitized default (`-apple-system, BlinkMacSystemFont, 'Segoe UI', ...`). Do not attempt to override.
- **Monospaced accent** — wrap project names or technical strings in `<samp>` or `` `code` `` for texture contrast. Use sparingly — no more than one monospaced element per logical line.

### Type scale (SVG sizes, in px at 1× render)

| Token                  | Size | Weight | Use                                    |
|------------------------|------|--------|----------------------------------------|
| `display-xl`           | 72   | 400    | Masthead wordmark "ALEX SHEN"          |
| `display-l`            | 48   | 400    | Section-break wordmarks (if any)       |
| `label-kicker`         | 12   | 500    | Small-caps subtitle under hairline     |
| `stamp`                | 11   | 500    | Issue / vol / date metadata            |

### Tracking & leading

- Display serif: `letter-spacing: 0.01em` (tight).
- Kickers & stamps: `letter-spacing: 0.22em` (wide, tracked uppercase).
- No italic display type. Italic is reserved for the single tagline ("*Building tools to detect AI-powered fraud.*") to preserve its status as the one editorial flourish.

## 3. Color

Muted, ink-on-bone palette. One warm accent used at ≤5% of visual area. Must work in **both** GitHub light and dark modes — we ship paired SVGs per asset and swap via `<picture>`.

### Tokens

| Token          | Light mode | Dark mode | Use                               |
|----------------|------------|-----------|-----------------------------------|
| `--ink`        | `#121212`  | `#ECE8E0` | Display type, hairlines           |
| `--ink-muted`  | `#5A5A5A`  | `#9A9690` | Stamps, kickers, secondary rules  |
| `--bone`       | `#FFFFFF`  | `#0D1117` | SVG background (matches GitHub)   |
| `--accent`     | `#B8301C`  | `#D9533F` | Exactly one element per asset     |

**Rules**:
- `--accent` appears once, small: the monogram mark, the issue-stamp bullet, or the masthead rule endcap. Never on headings, never on links (GitHub controls link color).
- Asset backgrounds MUST match GitHub's pane color (`#FFFFFF` light / `#0D1117` dark) so SVGs appear to float seamlessly. Do not use a contrasting fill.
- Hairlines: `stroke="currentColor"` where possible; else `--ink-muted` at `stroke-width="0.6"`.

### Dark-mode delivery pattern

```html
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="assets/masthead-dark.svg">
  <img src="assets/masthead-light.svg" alt="Alex Shen" width="680">
</picture>
```

## 4. Spacing & Rhythm

Markdown rhythm is coarse — we work with what GitHub gives us plus explicit `<br>` breaks.

### Vertical rhythm (top-to-bottom)

| Step | Mechanism                      | Use                                      |
|------|--------------------------------|------------------------------------------|
| XS   | single `<br>`                  | Within a list of projects                |
| S    | blank line                     | Paragraph break                          |
| M    | `<br><br>`                     | Subsection break                         |
| L    | `<hr>` or hairline SVG         | Major section change                     |
| XL   | masthead SVG's internal padding | Above the fold only                    |

**Rule of three**: never stack more than three `<br>` in a row. If you need more separation, insert a hairline.

### Horizontal alignment

- Entire README stays in `<div align="center">` — preserves current layout and reads as "masthead-centered".
- One exception: if a section grows beyond ~8 items, break it into a 2-column `<table>` with `align="center"` on the `<table>` but left-aligned cells. Do not mix left and center alignment inline — it reads as broken.

### Inline spacing

- Between inline items on one line: ` · ` (middle dot, spaces both sides) — editorial separator.
- Em dashes (`—`, U+2014) for descriptions: `[Project] — one-line description`. Never hyphen-minus.
- No emoji anywhere. Section meaning is carried by kickers, numerals, and rules.

## 5. Ornament & Components

Five canonical components. Each lives as one or two SVG files in `assets/`.

### 5.1 Masthead `assets/masthead-{light,dark}.svg`

Anchors the top of the page.

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   VOL · I     ALEX SHEN     MMXXVI               │
│   ──────────────────────────────────────         │
│   Founding Engineer · Scam AI · Berkeley SkyDeck │
│                                                  │
└──────────────────────────────────────────────────┘
```

- Canvas: `viewBox="0 0 680 200"`. Rendered at `width="680"`.
- Wordmark: `display-xl`, ink, centered, tracked `0.01em`.
- Flanking stamps: `stamp` tokens in `--ink-muted`, mono, uppercase, tracked `0.18em`. Left: `VOL · I`. Right: roman-numeral year `MMXXVI`.
- Hairline: single `<line>`, `stroke="currentColor"`, `stroke-width="0.6"`, `opacity="0.4"`. Spans full canvas minus 64px side margins.
- Subtitle line: default SVG `<text>`, `label-kicker` size, centered, letter-spaced.
- **The one accent use**: a 6×6 filled square in `--accent` placed as the endcap on one side of the hairline. No other color on the page should repeat this.

### 5.2 Section Kicker (markdown)

A small-caps label preceded by a numeral sign. Pure markdown — no SVG needed.

```markdown
<samp>§ 01 — AI & AUTOMATION</samp>
```

Rules:
- Numbered `§ 01`, `§ 02`, `§ 03` across the page. Never reset, never skip.
- En-space between `§` and the digits: `§&ensp;01`.
- All-caps section name. Preceded by em-dash `—`.
- Wrapped in `<samp>` for subtle monospaced texture — contrasts the serif masthead.

### 5.3 Project Row (markdown)

```markdown
<a href="URL">project-name</a> — one-line description · [meta tag] · <a href="SOURCE">source</a>
```

- Project name is lowercase (matches the repos' naming convention), wrapped in `<a>`.
- Em-dash, then description in sentence case, verb-led. No marketing adjectives.
- Optional meta chain separated by middle dots.
- Source link last, lowercase word "source". Only when there's a visible deploy URL AND a source link — otherwise omit the word.

### 5.4 Hairline Rule `assets/rule.svg` (or markdown `<hr>`)

- For in-section: native markdown `<hr>` is fine.
- For major transitions: `assets/rule.svg` — 680×24 canvas, single `<line>` at y=12, `stroke-width="0.6"`, `stroke="currentColor"`, opacity 0.4. Delivered via `<picture>` if we want the dark variant (in this case one file is fine — `currentColor` adapts — but GitHub's image rendering does not inherit `currentColor`, so ship both).

### 5.5 Colophon `assets/colophon.svg` (optional, signature move)

A footer stamp echoing the masthead. One line, mono, uppercase, `--ink-muted`.

```
SET · UI-SERIF & UI-MONO · SHIPPED FROM BERKELEY, CA · LAST REVISED ⟨DATE⟩
```

- Auto-updatable date is a stretch goal (GitHub Actions). For v1, ship it static, bump on major edits.

## 6. Voice & Copy Rules

The visual system sets a tone of restraint; copy must match.

- **Verb-led project descriptions**. "building X" not "a system that helps users with X".
- **One tagline, italicized**. Currently: *Building tools to detect AI-powered fraud.* This is the only italic in the whole document.
- **No adjectives of self-evaluation**: `passionate`, `experienced`, `skilled`, `innovative` — all banned.
- **Sentence case** for descriptions. **ALL CAPS (TRACKED)** for section kickers and stamps only.
- **Em-dashes, not hyphens**, to separate name and description.
- **No emoji**. Anywhere.
- **Links use the project's canonical name**. Not "here", not "my site".

## 8. Verification Rules (goes into `VERIFY.md`)

Checkpoints any README change must pass before push:

1. All `<a href>` URLs resolve (2xx/3xx).
2. All referenced SVG assets exist at `assets/*.svg` and are valid XML.
3. Both light and dark variants exist for every `<picture>` element.
4. Section numerals `§ 01`, `§ 02`, `§ 03` are sequential with no gaps.
5. Accent color `#B8301C` / `#D9533F` appears exactly once per asset.
6. No inline `<svg>`, `<style>`, `<script>` tags (would be stripped anyway; catch at author time).
7. README line count stays within 200–300 per global CLAUDE.md rule.

---

## 9. Out of Scope (v1)

- Dynamic GitHub stats / contribution graphs — conflicts with the static-publication metaphor.
- Any third-party image services (shields.io, etc.) — they break the typographic consistency.
- Animated SVGs — editorial minimal rejects motion-for-its-own-sake. If ever added, it should be one subtle ornament, never the masthead.
